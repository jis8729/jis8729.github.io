Case Study 01

이기종 설비 통합 Adapter 미들웨어 설계·개발

ACS-설비 간 직접 연동의 복잡성을 표준화된 미들웨어로 해결

표준화
인터페이스 구조
2인 팀 · Adapter 미들웨어 설계·개발 담당 제닉스 로보틱스 현대자동차 울산 의장 3공장 Java Spring Framework Delphi TCP/IP Modbus PostgreSQL MSSQL

Business Problem

울산 의장 3공장의 원키트 시스템에서 AGV, PLC, WMS 등 다양한 설비가 각각 상이한 프로토콜과 데이터 포맷을 사용하고 있었습니다. ACS가 모든 설비와 직접 연동해야 했기 때문에 통신 복잡도가 높았고, 신규 설비 추가 시마다 ACS 코드를 수정해야 하는 구조적 문제가 있었습니다.

What I Did

  • 이기종 소스의 프로토콜과 데이터 포맷을 분석하여 ACS 표준 규격으로 변환·통합하는 Adapter 미들웨어 설계
  • 벤더별 상이한 통신 방식(TCP/IP, Modbus 등)을 공통 인터페이스로 추상화
  • ACS는 미들웨어의 표준 인터페이스만 호출하면 되는 구조로 전환

Business Problem

공장 레이아웃이 변경될 때마다 AGV 경로를 수작업으로 수정해야 했고 이 작업에 2일 이상 소요되었습니다. 그 사이 생산 라인이 멈추거나 비효율적으로 운영되었고, 평균 사이클 타임이 목표를 초과하는 문제가 있었습니다.

What I Did

  • 하드코딩 경로 계획을 그래프 기반 동적 경로 탐색으로 전환, Priority-Based Search 적용
  • 거리·시간 기반 비용 함수와 레이아웃 제약 조건 추가
  • Repository Pattern 도입으로 제어 로직·데이터 접근·PLC 통신 간 결합도 분리

Business Problem

IoT 디바이스마다 통신 프로토콜이 달라 신규 장비 도입 시마다 개발이 병목이 되었습니다. 근본 원인은 모든 프로토콜 파싱 로직이 하나의 메서드에 집중된 if-else 구조로, 확장성을 고려하지 않은 초기 설계와 누적된 기술 부채였습니다.

What I Did

  • 프로토콜 파싱 로직을 독립적인 Strategy 클래스로 분리
  • Factory가 요청에 맞는 Strategy를 동적으로 선택하도록 설계 (OCP, SRP 준수)
  • RabbitMQ 기반 메시지 큐 연동 및 Redis 캐싱으로 실시간 데이터 처리 파이프라인 구축
